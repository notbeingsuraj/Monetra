# Monetra – Design Requirements Document (DRD)

## 1. Design Philosophy

Monetra should feel like a **modern fintech product**, not a generic mobile application.

Design principles:

• minimal interface
• clear hierarchy
• strong typography
• functional simplicity

The UI should emphasize **trust, clarity, and reliability**.

---

## 2. Visual Style

### Color Palette

Primary
Charcoal (#1C1C1C)

Background
Warm White (#F5F5F3)

Accent
Muted Emerald (#1F7A63)

Secondary
Neutral Gray (#8C8C8C)

Avoid bright gradients or saturated colors.

---

## 3. Typography

Recommended fonts:

• Inter
• SF Pro (iOS)
• Roboto (Android)

Text hierarchy:

Heading
Bold large text

Subheading
Medium weight

Body
Regular weight

---

## 4. Layout Principles

Spacing system:

8px grid system.

Key rules:

• large padding
• consistent margins
• balanced whitespace

---

## 5. Navigation

Primary navigation uses bottom tabs.

Tabs:

Dashboard
Loans
Add Loan
History
Profile

---

## 6. Core Screens

### 6.1 Dashboard

Displays:

• Trust Score Card
• Money Lent Summary
• Money Borrowed Summary
• Active Loans

---

### 6.2 Add Loan Screen

Simple form with fields:

Borrower
Amount
Due Date
Note

Primary button:

“Record Loan”

---

### 6.3 Loan History

List view showing:

• borrower name
• amount
• status
• due date

---

### 6.4 Loan Detail Screen

Displays:

• loan amount
• borrower name
• due date
• repayment status

---

### 6.5 Profile

Shows:

• user name
• trust score
• total loans
• settings

---

## 7. Interaction Design

Animations should be minimal and purposeful.

Examples:

• smooth screen transitions
• confirmation feedback
• subtle micro-interactions

---

## 8. Accessibility

The design should include:

• readable font sizes
• sufficient color contrast
• simple interaction flows
